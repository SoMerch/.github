# Security Policy

SoMerch takes security and responsible disclosure seriously.

If you believe you have found a security issue related to SoMerch public repositories, documentation, platform behavior, app links, or integration examples, please report it privately.

---

## Reporting a security issue

Please do not open a public GitHub issue for security vulnerabilities.

Instead, contact us at:

[support@somerch.co](mailto:support@somerch.co)

Use this subject line:

```text
Security Report: SoMerch
```

Please include as much detail as possible:

- affected repository, page, app, or workflow
- description of the issue
- steps to reproduce
- potential impact
- screenshots or logs, if safe to share
- your contact information for follow-up

---

## What not to include

Please do not send:

- real customer addresses
- real employee personal data
- API keys, tokens, or credentials
- full database exports
- private customer files
- sensitive information unrelated to the report

If sensitive data is necessary to explain the issue, redact it first.

---

## Public repositories

The public SoMerch GitHub repositories are intended for documentation, product data examples, public roadmap information, brand assets, and integration concepts.

They should not contain:

- production credentials
- customer data
- employee addresses
- private business contracts
- internal infrastructure details
- unpublished product source code
- font files or private licensed assets

---

## Response expectations

We aim to review security reports as quickly as possible.

Response time may vary depending on the severity, clarity, and complexity of the report.

---

## Responsible disclosure

Please give SoMerch reasonable time to investigate and respond before publicly discussing a potential vulnerability.

We appreciate responsible reports that help keep SoMerch safe for customers, partners, and users.
