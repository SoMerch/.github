# Contributing

Thank you for your interest in improving SoMerch public resources.

The public SoMerch GitHub repositories are mainly used for documentation, public roadmap feedback, product data examples, brand assets, and integration concepts.

---

## What contributions are welcome

Useful contributions include:

- fixing broken links
- improving documentation clarity
- reporting outdated public information
- suggesting missing workflow documentation
- improving product data schema examples
- suggesting integration use cases
- reporting typos or formatting issues

---

## What contributions are not accepted publicly

Please do not submit:

- customer data
- employee addresses
- private order details
- production credentials
- API keys or tokens
- private source code
- copyrighted files without permission
- font files
- internal business documents
- confidential procurement information

---

## Before opening an issue

Before opening an issue, check whether:

- the issue is public-safe
- it does not include private customer data
- it belongs in GitHub rather than customer support
- the relevant repository is selected

For product support, contact:

[support@somerch.co](mailto:support@somerch.co)

---

## Pull requests

Pull requests should be focused and easy to review.

Good pull requests usually:

- fix one clear issue
- explain the reason for the change
- avoid unrelated edits
- keep wording consistent with SoMerch positioning
- avoid unsupported claims

---

## Brand wording

Use the correct brand name:

**SoMerch**

Avoid:

- Somerch
- SO Merch
- So Merch
- somerch as the brand name

---

## Communication style

Keep communication clear, practical, and professional.

SoMerch public resources should communicate operational clarity, trust, and usefulness for distributed teams managing corporate merch.
