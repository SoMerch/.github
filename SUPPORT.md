# Support

This document explains how to contact SoMerch and how to use the public GitHub repositories.

---

## Product and customer support

For product questions, demos, managed merch programs, orders, fulfillment, or customer support, contact SoMerch through:

- Website: [somerch.co](https://somerch.co)
- Email: [support@somerch.co](mailto:support@somerch.co)

---

## Public GitHub repositories

The SoMerch public GitHub repositories are mainly used for:

- public roadmap information
- customer-facing documentation
- developer-oriented API concepts
- product data schemas
- brand assets
- integration examples
- public feedback

They are not the main support channel for active orders, private customer data, invoices, shipment claims, or production issues.

---

## When to open a GitHub issue

You can open a GitHub issue for:

- documentation feedback
- broken links
- unclear wording
- public roadmap feedback
- product data schema suggestions
- integration questions
- brand asset corrections

---

## When not to open a GitHub issue

Do not open a public GitHub issue for:

- active customer orders
- shipment address changes
- invoice questions
- private procurement information
- security vulnerabilities
- personal data
- confidential company details
- production credentials

For these cases, contact SoMerch privately through the website or support email.

---

## Security reports

Security reports should be sent privately.

See [SECURITY.md](SECURITY.md) for reporting instructions.
